# 1. Введение
Данный раздел содержит общую информацию о программном обеспечении «Автоматизированная система управления запасами для розничного магазина» (далее — АСУЗРМ), определяет цели и границы применения, включает описание используемой терминологии и ссылки на нормативные документы.

## Цели
Цель настоящего документа — формализовать требования к разработке программного обеспечения АСУЗРМ. 
Документ предназначен для использования:
   - Заказчиком, для проверки соответствия функциональности системы бизнес-требованиям;
   - Разработчиком, для проектирования, реализации и тестирования системы;
   - Тестировщиком, для построения планов и сценариев тестирования;
   - Сопровождающим персоналом, для эксплуатации и дальнейшего развития системы.

Назначение документа — обеспечить однозначное понимание всех требований к системе, включая функциональные, нефункциональные, интерфейсные и эксплуатационные характеристики, а также определить ограничения и предположения, влияющие на разработку и внедрение.

## Границы применения
Автоматизированная система управления запасами для розничного магазина (АСУЗРМ) предназначена для автоматизации процессов учёта, контроля и анализа товарных запасов.
Программное обеспечение охватывает следующие функциональные области:
   - учёт поступлений товаров от поставщиков;
   - учёт продаж и автоматическое списание товаров;
   - контроль остатков и уведомление о достижении пороговых значений;
   - формирование аналитических отчётов по движению товаров;
   - поддержка интеграции с кассовыми терминалами и системой бухгалтерского учёта.

АСУЗРМ не включает в себя:
   - функции финансового учёта вне контекста товарных операций;
   - управление персоналом;
   - CRM-функциональность.

Система предназначена для эксплуатации в розничных торговых точках малого и среднего масштаба, функционирующих в локальной сети или облачной инфраструктуре.

## Термины и аббревиатуры
**АСУЗРМ** - Автоматизированная система управления запасами для розничного магазина.   
**SKU - Stock Keeping Unit** — уникальный идентификатор товарной позиции.   
**API - Application Programming Interface** — интерфейс взаимодействия программных компонентов.  
**GUI	- Graphical User Interface** — графический пользовательский интерфейс.   
**БД** - База данных.   
**Пользователь** - Сотрудник магазина, взаимодействующий с системой.  
**Отчёт** - Сформированный системой документ, содержащий аналитическую информацию по движению товаров.

## Краткий обзор
Настоящий документ разработан в соответствии с требованиями стандарта [IEEE-830].
Документ включает следующие разделы:
  - Раздел 1. Введение — содержит общие сведения о системе и назначении документа;
  - Раздел 2. Общее описание — определяет контекст системы, её функции, пользователей, ограничения и допущения;
  - Раздел 3. Специфические требования — описывает функциональные и нефункциональные требования, интерфейсы, форматы данных и условия эксплуатации;
