# 2. Общее описание

## Описание изделия
- **Интерфейсы системы**: подлежат выяснению.
- **Интерфейсы пользователя**: Минимальный пользовательский интерфейс для загрузки данных (каса, справочники, прайсы), остальные операции автоматизированы.
- **Интерфейсы аппаратных средств ЭВМ**: Компьютер с операционной системой windows, установленое приложение, работающее в локальной сети.
- **Интерфейсы программного обеспечения**: Импорт данных из файлов Z-отчетов, приходных накладных, прайсов; взаимодействие с базой данных.
- **Интерфейсы коммуникаций**: Локальная файловая система или внутренняя сеть.
- **Ограничения памяти**: Не предъявляются особо строгие требования, учитывая простоту приложения.
- **Действия**: Автоматическое отслеживание уровней запасов, формирование и отправка заказов, загрузка данных.

## Функции изделия
- Автоматический мониторинг запасов товаров.
- Формирование заказов при достижении неснижаемого уровня.
- Рассылка автозаказов поставщикам.
- Учет цен и условий поставки.
- Выбор поставщика с учетом цены и сроков доставки.
- Обеспечение работы системы в автономном режиме, с автоматическим восстановлением после сбоев.
- Возможность загрузки данных вручную через интерфейс.
- Генерация отчетов для владельца магазина.

## Характеристики пользователей
- **Администратор**: 
   - IT-специалист с глубокими знаниями.
   - Выполняют функции: настройка параметров системы, а также управление ролями, настройками справочников, загрузкой прайс-листов.
- **Оператор НСИ**:
   - Сотрудник, обладающий базовыми навыками работы с ЭВМ.
   - Выполняют функции: ввод и обновление данных о поставщиках, прайсах.
- **Руководитель/владелец магазина**:
   - Обладает базовыми навыками работы с ЭВМ.
   - Выполняют функции: просмотр текущих остатков, планов поставок, отчетов.

## Ограничения
- Работа системы — локальная, без внешних интерфейсов.
- Время восстановления системы после сбоя — до 4 часов.
- Производительность — приложение работает на компьютере с операционной системой Windows.
- Работа в изолированной среде или в сети, не требующей высокой пропускной способности.
- Обмен данными происходит через файлы или внутреннюю базу данных, обновляемые регулярно.
- Пользовательская аутентификация — по логину и паролю.

## Предположения и зависимости
- Данные с кассы и приходных накладных поступают регулярно и в стандартизированном формате.
- Поставщики предоставляют прайсы и даты поставок, которые система использует для оптимизации заказов.
- В системе отсутствует возможность ручного воздействия оператора на автоматический механизм формирования заказов.
- Все необходимые справочники, прайсы и документы вводятся и обновляются через интерфейс загрузки данных.
- Вся логика работы системы — автоматическая, без необходимости вмешательства оператора после загрузки данных.
- Время формирования автозаказа — не более 30 минут после запуска системы утром.
- Время восстановления системы после сбоя — до 4 часов.
- В системе реализована многопользовательская аутентификация по логину и паролю с разграничением ролей.
